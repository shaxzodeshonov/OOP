Week 1 · Lesson 1

OOP Fundamentals

Object-Oriented Programming organizes code around objects — bundles of data and behavior — instead of just functions and logic.

4 Core Concepts

Object

A specific instance. myCar is an object of type Car. It has real data in memory.

Class

A blueprint/template. Defines what data and behaviors all objects of that type will have.

Attribute (Data Member)

Variables that belong to a class. E.g., a Car has speed, color, brand.

Behavior (Member Function)

Functions that belong to a class. E.g., accelerate(), brake(), getSpeed().

Why OOP?

  • Groups related data + functions together → cleaner code
  • Encapsulation hides internal details → less bugs
  • Reusable via inheritance and polymorphism (coming in later courses)
  • Models real-world things naturally
Week 1 · Lesson 2

Defining Classes & Creating Objects

Class Syntax

// Class definition (blueprint)
class Car {
public:
    // Data members (attributes)
    string brand;
    int    speed;

    // Member function (behavior)
    void accelerate(int amount) {
        speed += amount;
    }

    void display() {
        cout << brand << " going " << speed << " km/h";
    }
}; // ← semicolon required!

C++

Creating & Using Objects

int main() {
    Car myCar;           // create object (instance)
    myCar.brand = "BMW"; // access member with dot (.)
    myCar.speed = 0;

    myCar.accelerate(60); // call member function
    myCar.display();      // output: BMW going 60 km/h

    Car car2;            // each object is independent!
    car2.brand = "Toyota";
    car2.speed = 100;
    return 0;
}
C++
⚠ Key Rule

Every class definition ends with }; — the semicolon after the closing brace is mandatory. Forgetting it causes confusing compiler errors.

Week 1 · Lesson 3

Member Functions

Member functions define what objects do. They can be defined inside the class (inline) or outside using the scope resolution operator ::.

Inside vs Outside Definition

class Rectangle {
public:
    double width, height;

    // Defined INSIDE class (inline)
    double area() {
        return width * height;
    }

    // Only declared inside, defined OUTSIDE
    double perimeter();
};

// Outside definition uses ClassName:: prefix
double Rectangle::perimeter() {
    return 2 * (width + height);
}
C++

Functions With Parameters

class BankAccount {
public:
    double balance;

    void deposit(double amount) {  // parameter
        balance += amount;
    }

    bool withdraw(double amount) {
        if (amount > balance) return false;
        balance -= amount;
        return true;
    }
};
C++
Week 1 · Lesson 4

Encapsulation, Getters & Setters

Encapsulation = hiding data (making it private) and controlling access through public getter/setter functions.

Access Specifiers

public

Accessible from anywhere — inside class, outside class, everywhere.

private

Only accessible inside the class itself. Default for class keyword.

Getters & Setters with Validation

class Student {
private:
    string name;
    int    grade; // hidden from outside

public:
    // SETTER — with validation
    void setGrade(int g) {
        if (g >= 0 && g <= 100)
            grade = g;
        else
            cout << "Invalid grade!\n";
    }

    // GETTER — read-only access
    int getGrade() const {
        return grade;
    }

    void setName(string n) { name = n; }
    string getName() const { return name; }
};

int main() {
    Student s;
    s.setGrade(150); // prints "Invalid grade!"
    s.setGrade(88);  // works fine
    cout << s.getGrade(); // 88
    // s.grade = 88;  ← ERROR! private
}
C++
Week 1 · Lesson 5

Separating Interface from Implementation

In professional C++, classes are split across two files: a header (.h) for the interface, and a source (.cpp) for the implementation.

Student.h — Header File (Interface)

// Student.h — what the class LOOKS like
#ifndef STUDENT_H   // include guard
#define STUDENT_H

#include <string>
using namespace std;

class Student {
private:
    string name;
    int grade;
public:
    void setName(string n);
    string getName() const;
    void setGrade(int g);
    int getGrade() const;
};

#endif
Student.h

Student.cpp — Source File (Implementation)

// Student.cpp — HOW it works
#include "Student.h"
#include <iostream>

void Student::setName(string n) { name = n; }
string Student::getName() const { return name; }
void Student::setGrade(int g) {
    grade = (g >= 0 && g <= 100) ? g : 0;
}
int Student::getGrade() const { return grade; }
Student.cpp
📌 Include Guard

#ifndef / #define / #endif prevents the header from being included twice. Always add these to every .h file you write.

Week 2 · Lesson 1

Class Scope & Access Functions

Class Scope

All members of a class exist in class scope — member functions can access all data members directly (no object prefix needed inside the class).

class Circle {
private:
    double radius; // class scope
public:
    double area() {
        return 3.14 * radius * radius; // direct access
    }
};
C++

Access vs Utility Functions

Access Functions

Public functions that read (get) or write (set) private data. Exposed to users of the class.

Utility Functions

Private helper functions used internally by other member functions. Not part of the public interface.

class Payment {
private:
    double amount;
    bool isValid(double a) { // utility (private)
        return a > 0 && a < 10000;
    }
public:
    void setAmount(double a) { // access (public)
        if (isValid(a)) amount = a; // uses utility
    }
    double getAmount() const { return amount; }
};
C++
Week 2 · Lesson 2

Constructors & Destructors

Constructors auto-run when an object is created. Destructors auto-run when an object is destroyed.

Constructor Rules

  • Same name as the class
  • No return type (not even void)
  • Can be overloaded (multiple constructors)
  • Default constructor has no parameters

Constructor Types

class Person {
private:
    string name;
    int    age;
public:
    // 1. Default constructor
    Person() {
        name = "Unknown";
        age = 0;
        cout << "Person created\n";
    }

    // 2. Parameterized constructor
    Person(string n, int a) {
        name = n;
        age  = a;
    }

    // 3. Destructor (~ prefix, no params)
    ~Person() {
        cout << name << " destroyed\n";
    }

    void print() { cout << name << ", age " << age; }
};

int main() {
    Person p1;              // calls default ctor
    Person p2("Alice", 30); // calls parameterized
    p2.print();             // Alice, age 30
} // destructors called here automatically
C++

Member Initializer List (preferred)

Person(string n, int a) : name(n), age(a) {
    // body can be empty — members initialized above
}
C++
Week 2 · Lesson 3

Const Objects & Composition

Constant Member Functions

The const keyword after a function signature promises the function won't modify any data members. Required to call on const objects.

class Temperature {
private: double celsius;
public:
    Temperature(double c) : celsius(c) {}

    double toFahrenheit() const { // const — won't modify
        return celsius * 9.0/5.0 + 32;
    }
    void setTemp(double c) { celsius = c; } // NOT const
};

const Temperature freezing(0.0);
// freezing.setTemp(5);  ← ERROR — const object!
cout << freezing.toFahrenheit(); // OK — const function
C++

Composition (Object inside Object)

Composition = a class has another class as a data member. "Has-a" relationship.

class Date {
public:
    int day, month, year;
    Date(int d, int m, int y) : day(d), month(m), year(y) {}
};

class Employee {
private:
    string name;
    Date birthDate; // composed Date object
public:
    Employee(string n, Date d) : name(n), birthDate(d) {}
    void print() {
        cout << name << " born: "
             << birthDate.day << "/" << birthDate.month;
    }
};
C++
Week 2 · Lesson 4

friend, Pointers & Dynamic Memory

friend Functions

A friend function is a non-member that's granted access to private members. Declared inside the class with keyword friend.

class Box {
private: double volume;
public:
    Box(double v) : volume(v) {}
    friend void printVolume(Box b); // friend declaration
};

void printVolume(Box b) { // can access private!
    cout << b.volume; // OK — friend
}
C++

Pointers to Objects

Student s("Ali", 90);
Student* ptr = &s;      // pointer to object

ptr->setGrade(95);     // arrow -> for pointer access
cout << ptr->getGrade(); // 95

// Same as: (*ptr).getGrade()
C++

Dynamic Memory: new & delete

// Allocate on the HEAP (lives until deleted)
Student* p = new Student("Bob", 85);
p->print();

delete p;   // MUST delete to free memory!
p = nullptr; // good practice

// Array of objects
Student* arr = new Student[5];
delete[] arr; // use delete[] for arrays
C++
⚠ Memory Leak

Every new must have a matching delete. Forgetting causes memory leaks — your program keeps consuming RAM silently.

Week 2 · Lesson 5

Static Data Members & Functions

Static members belong to the class, not to any specific object. Shared across all instances.

Static Example — Object Counter

class Widget {
private:
    static int count; // shared by ALL widgets
    int id;
public:
    Widget() { id = ++count; }
    ~Widget() { --count; }

    static int getCount() { return count; }
    int getId() const { return id; }
};

// MUST define static outside class:
int Widget::count = 0;

int main() {
    Widget a, b, c;
    cout << Widget::getCount(); // 3 — class-level call
    cout << a.getId();           // 1
}
C++
📌 Remember

Static member functions can ONLY access static data members — they have no this pointer. Call them using ClassName::functionName().

Week 3–4 · Lesson 1

Operator Overloading

Operator overloading lets you define what +, ==, [], etc. mean for your custom classes.

Why Overload Operators?

// Without overloading — ugly:
result = v1.add(v2);

// With overloading — natural:
result = v1 + v2;
C++

As Member Function

class Vector2D {
public:
    double x, y;
    Vector2D(double x, double y) : x(x), y(y) {}

    // Member: left operand is implicit (this)
    Vector2D operator+(const Vector2D& rhs) const {
        return Vector2D(x + rhs.x, y + rhs.y);
    }

    bool operator==(const Vector2D& rhs) const {
        return x == rhs.x && y == rhs.y;
    }
};

int main() {
    Vector2D a(1,2), b(3,4);
    Vector2D c = a + b;    // c.x=4, c.y=6
    cout << (a == b);    // 0 (false)
}
C++

As Non-Member (Global) Function

// When left operand isn't your class (e.g., int + MyClass)
Vector2D operator+(double scalar, const Vector2D& v) {
    return Vector2D(scalar + v.x, scalar + v.y);
}
// Used like: result = 5.0 + myVector;
C++
Week 3–4 · Lesson 2

Stream Operators << and >>

Overloading << (insertion) and >> (extraction) lets you use cout and cin with your objects naturally.

Rules for Stream Operators

  • Must be non-member functions (left operand is ostream, not your class)
  • Take stream by reference, return stream by reference (for chaining)
  • Typically declared as friend to access private data

Implementation

class Point {
private:
    int x, y;
public:
    Point(int x, int y) : x(x), y(y) {}

    // Declare friends so they can access x, y
    friend ostream& operator<<(ostream& out, const Point& p);
    friend istream& operator>>(istream& in, Point& p);
};

// Output: cout << myPoint
ostream& operator<<(ostream& out, const Point& p) {
    out << "(" << p.x << ", " << p.y << ")";
    return out; // allows: cout << p1 << p2
}

// Input: cin >> myPoint
istream& operator>>(istream& in, Point& p) {
    in >> p.x >> p.y;
    return in;
}

int main() {
    Point p(3, 7);
    cout << p;            // prints: (3, 7)
    cout << p << "\n";   // chaining works!
}
C++
Week 3–4 · Lesson 3

Unary Operator Overloading

Unary operators take only one operand: -, !, ++, --.

Prefix vs Postfix Increment

class Counter {
private: int val;
public:
    Counter(int v = 0) : val(v) {}

    // Prefix ++c (no dummy param)
    Counter& operator++() {
        ++val;
        return *this;
    }

    // Postfix c++ (dummy int param)
    Counter operator++(int) {
        Counter old = *this; // save copy
        ++val;
        return old;         // return original
    }

    // Unary minus
    Counter operator-() const {
        return Counter(-val);
    }

    int get() const { return val; }
};

int main() {
    Counter c(5);
    cout << (++c).get(); // 6  (prefix)
    cout << (c++).get(); // 6  (postfix returns old)
    cout << c.get();     // 7  (now incremented)
    cout << (-c).get(); // -7
}
C++
📌 Trick to Remember

Postfix operator++(int) has a dummy int parameter — just to distinguish it from prefix. You never pass a value for it.
